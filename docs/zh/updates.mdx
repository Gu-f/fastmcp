FastMCP 2.9 是一个重大版本，引入了两个超越基本 MCP 协议的重要功能。

🤝 *MCP 中间件* 带来了一个灵活的中间件系统，用于拦截和控制服务器操作 - 包括认证、日志记录、速率限制和自定义业务逻辑，无需触及核心协议代码。

✨ *服务器端类型转换* 用于提示解决了开发者的一个主要痛点：虽然 MCP 需要字符串参数，但您的函数现在可以使用原生 Python 类型如列表和字典，自动转换处理复杂性。

这些功能将 FastMCP 从简单的协议实现转变为构建复杂 MCP 应用程序的强大框架。结合新的 `File` 二进制数据工具以及认证和序列化的改进，此版本使 FastMCP 在保持完整协议合规性的同时变得更加灵活和开发者友好。
</Card>
</Update>

<Update label="FastMCP 2.8" description="2025年6月11日" tags={["发布", "博客文章"]}>
<Card 
title="FastMCP 2.8: 变换升级" href="https://www.jlowin.dev/blog/fastmcp-2-8-tool-transformation" 
img="https://www.jlowin.dev/_image?href=%2F_astro%2Fhero.su3kspkP.png&w=1000&h=500&f=webp" 
cta="阅读更多"  
>
FastMCP 2.8 已发布，这个版本专注于让您完全掌控您的工具。

此版本为打造完美的 LLM 体验提供了众多新功能：

🛠️ 工具变换

标题功能让您可以包装任何工具——来自您自己的代码、第三方库或 OpenAPI 规范——创建增强的、LLM 友好的版本。您可以重命名参数、重写描述和隐藏参数，而无需触及原始代码。

此功能是与 Bill Easton 密切合作开发的。正如 Bill 精辟地[所说](https://www.linkedin.com/posts/williamseaston_huge-thanks-to-william-easton-for-providing-activity-7338011349525983232-Mw6T?utm_source=share&utm_medium=member_desktop&rcm=ACoAAAAd6d0B3uL9zpCsq9eYWKi3HIvb8eN_r_Q)："工具变换颠覆了提示工程：停止编写工具友好的 LLM 提示，开始提供 LLM 友好的工具。"

🏷️ 组件控制

既然您在变换工具，就需要一种隐藏旧工具的方法！在 FastMCP 2.8 中，您可以程序化地启用/禁用任何组件，对于所有一直在问 FastMCP 标签用途的人——它们终于有用处了！您现在可以使用标签声明性地过滤向客户端暴露哪些组件。

🚀 默认实用

最后，为了确保与生态系统的最大兼容性，我们做出了实用的决定，将所有 OpenAPI 路由默认为工具模式，使您的整个 API 立即可被任何使用工具的代理访问。当整个相关行业跟上步伐并支持相关资源时，我们将恢复原来的默认设置——但在 OpenAI、Anthropic 或 Google 实现之前，您无需做任何额外工作！

</Card>
</Update>

<Update label="FastMCP 2.7" description="2025年6月6日" tags={["发布"]}>
<Card 
title="FastMCP 2.7: 精简编程" href="https://github.com/jlowin/fastmcp/releases/tag/v2.7.0" 
img="/assets/updates/release-2-7.png"
cta="阅读发布说明"  
>
FastMCP 2.7 已发布！

最值得注意的是，它引入了备受期待的（也是 Pythonic 的）"naked"装饰器用法：

```python {3}
mcp = FastMCP()

@mcp.tool
def add(a: int, b: int) -> int:
    return a + b
```

此外，装饰器现在返回它们创建的对象，而不是被装饰的函数。这是一个重要的可用性增强。

更新的主要部分专注于改进 FastMCP 内部结构，包括对私有 API 的一些破坏性内部更改。自 1.0 以来一直保留的许多函数现在已被弃用。
</Card>
</Update>
